### Variants

Six visual variants ordered by emphasis. Use **one** `default` button per action group as the primary CTA. Pair it with `outline` or `ghost` for secondary actions.

<Canvas of={ButtonStories.AllVariants} />

<SectionDivider />

### Sizes

`lg` for hero sections. `default` for standard UI. `sm` for toolbars and filter bars. `xs` for inline tags. The `icon-*` sizes create square buttons â€” always add `aria-label`.

<Canvas of={ButtonStories.AllSizes} />

<SectionDivider />

### Icons

Icons reinforce meaning. Leading icons suit action verbs (`Download`, `Search`). Trailing icons suit direction (`â†’`, `â†“`). shadcn auto-sizes icons via `[&_svg]` â€” no manual size needed.

<Canvas of={ButtonStories.WithIcons} />

<SectionDivider />

### Destructive Actions

Use `variant="destructive"` only for irreversible operations. Always pair with an **AlertDialog** confirmation step â€” never a bare click.

### Loading Transition

The component is intentionally unopinionated about loading state. You control it with `disabled` + `Loader2` + a label change. This keeps the loading pattern composable and testable.

### Focus Ring

Focus rings use `focus-visible` â€” they appear on keyboard navigation only, never on mouse clicks. This is baked into the shadcn base styles. The `destructive` variant overrides to a red ring.
